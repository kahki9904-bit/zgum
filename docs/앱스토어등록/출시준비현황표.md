# Z:GUM 스토어 출시 준비 현황표

기준일: 2026-06-25

상태 기준:

| 상태 | 의미 |
|---|---|
| 준비됨 | 현재 저장소 안에 제출 또는 작업 근거가 있음 |
| 일부 준비 | 초안, 코드, 이미지 등은 있으나 제출 전 보완이 필요함 |
| 미준비 | 파일이나 설정이 없거나, 현재 상태로 제출할 수 없음 |
| 확인 필요 | 외부 계정, 콘솔, 실제 기기, URL 등 저장소만으로 확정할 수 없음 |
| 차단 | 이 항목이 해결되지 않으면 스토어 심사 또는 출시 빌드가 막힐 가능성이 큼 |

## 전체 요약

| 구분 | 현재 상태 | 판단 |
|---|---:|---|
| 앱 기본 코드/화면 | 일부 준비 | Flutter 앱 구조, 지도, 설정, 언어, 스크린샷 후보는 있음 |
| Android 출시 설정 | 차단 | 패키지명과 서명이 아직 출시용이 아님 |
| iOS 출시 설정 | 차단 | 번들 ID가 예시값이고 iOS 테스트 리포트에 긴급 이슈가 남아 있음 |
| 스토어 문서 | 일부 준비 | 문서 파일은 있으나 내용이 거의 비어 있고 일부 한글이 깨짐 |
| 개인정보/권한 설명 | 일부 준비 | 방향성은 있으나 제출용 문구와 URL이 필요함 |
| 스크린샷 | 일부 준비 | 후보 이미지는 있으나 스토어 규격 세트로 재가공 필요 |
| 아이콘 | 일부 준비 | 원본과 플랫폼 아이콘은 있으나 Google 512px 제출본 확인 필요 |
| 심사 계정/외부 콘솔 | 확인 필요 | Google Play Console, Apple Developer, Firebase/Google Maps 콘솔 상태는 저장소만으로 확인 불가 |

## 제출 준비 현황

| 항목 | 상태 | 현재 근거 | 부족한 것 | 우선순위 |
|---|---|---|---|---|
| 앱 이름 | 일부 준비 | `lib/core/constants.dart`는 `Z:GUM`, i18n도 `Z:GUM` 사용 | Android/iOS 표시명은 `ZGUM`이라 스토어명과 앱 내부 표기 통일 필요 | 높음 |
| Android 앱 ID | 차단 | `android/app/build.gradle.kts`의 `applicationId = "com.example.zgum"` | 출시용 고유 패키지명 필요. 예: `com.zgum.app` 같은 최종 ID | 매우 높음 |
| iOS 번들 ID | 차단 | `ios/Runner.xcodeproj/project.pbxproj`의 `PRODUCT_BUNDLE_IDENTIFIER = com.example.zgum` | Apple Developer에 등록할 최종 Bundle ID 필요 | 매우 높음 |
| Android 릴리즈 서명 | 차단 | `release` 빌드가 `signingConfigs.getByName("debug")` 사용 | 업로드 키스토어, `key.properties`, release signing 설정 필요 | 매우 높음 |
| 앱 버전 | 준비됨 | `pubspec.yaml`의 `version: 1.0.0+1` | 첫 제출용 버전으로 유지할지 최종 확인 | 중간 |
| 앱 아이콘 | 일부 준비 | `assets/icon/app_icon.png`, Android mipmap 아이콘, iOS AppIcon 세트 있음 | Google Play용 `512 x 512` PNG 제출본 별도 생성/확인 필요. 현재 원본은 `1254 x 1254` | 높음 |
| 피처 그래픽 | 미준비 | 저장소에서 `1024 x 500` 스토어 피처 그래픽 확인 안 됨 | Google Play 피처 그래픽 제작 필요 | 높음 |
| 앱 스크린샷 후보 | 일부 준비 | `docs/screenshots/00_splash.png`부터 `08_detail.png` 등 후보 존재 | 현재 주요 후보는 `720 x 1480`. Google/Apple 제출 규격별 최종 세트 재제작 필요 | 높음 |
| Google Play 휴대폰 스크린샷 | 일부 준비 | 후보 이미지는 8장 이상 있음 | 최소 2장 이상 제출 가능하지만, 권장 세트 6-8장을 `1080 x 1920` 이상으로 재캡처 또는 디자인 필요 | 높음 |
| App Store iPhone 스크린샷 | 일부 준비 | 후보 이미지는 있음 | 6.9인치 iPhone 규격용 스크린샷 필요. 현재 `720 x 1480`은 제출 규격으로 부족할 수 있음 | 높음 |
| 스토어 설명문 | 미준비 | `docs/앱스토어등록/스토어설명문.md` 파일은 있으나 제목 수준 | Google 짧은 설명, 긴 설명, Apple subtitle, description, keywords 작성 필요 | 높음 |
| 개인정보처리방침 | 일부 준비 | `docs/앱스토어등록/개인정보처리방침.md` 파일은 있으나 내용 부족 | 제출용 본문, 공개 URL, 데이터 삭제 요청 방법 필요 | 매우 높음 |
| 심사 제출 체크리스트 | 일부 준비 | `docs/앱스토어등록/심사제출체크리스트.md` 파일은 있으나 내용 부족 | Google/Apple 항목별 체크리스트 보강 필요 | 중간 |
| 한글 문서 인코딩 | 차단 | 여러 문서와 iOS `Info.plist` 권한 문구가 깨진 글자로 표시됨 | 제출 문서와 앱 권한 팝업 문구를 정상 한글로 복구 필요 | 매우 높음 |
| 위치 권한 설명 | 일부 준비 | Android 위치 권한, iOS 위치 사용 설명 키, `ConsentGuard` 위치 동의 화면 존재 | iOS 문구 깨짐. 앱 진입 흐름에서 `ConsentGuard`가 현재 연결되어 있는지 재확인 필요 | 매우 높음 |

| 카메라/사진 권한 | 일부 준비 | iOS 카메라/사진 설명 키 존재, 이미지 선택 기능 의존성 있음 | iOS 문구 깨짐. 실제 기능이 스토어 설명/개인정보 문서와 일치해야 함 | 높음 |
| 백그라운드 위치 | 확인 필요 | 현재 Android manifest에는 background location 권한 없음 | 이음/근접 알림을 백그라운드에서 제공할지 정책 확정 필요 | 높음 |
| Data Safety 작성 | 미준비 | 저장소에 Google Play Data safety 작성표 없음 | 위치, 푸시 토큰, 기기 ID, 사용 기록, 사진, 서버 저장 여부를 표로 정리 필요 | 매우 높음 |
| App Privacy 작성 | 미준비 | 저장소에 Apple App Privacy 작성표 없음 | 데이터 유형별 수집/연결/추적 여부 정리 필요 | 매우 높음 |
| 콘텐츠 등급 자료 | 미준비 | 별도 등급 질문 답변표 없음 | Google 콘텐츠 등급, Apple 연령 등급 답변 준비 필요 | 중간 |
| 테스트 계정/심사 접근 | 확인 필요 | 가입 없는 흐름으로 보이나 관리자/파트너 기능 존재 | 심사자가 접근할 수 있는 방법, 테스트 절차, 심사 메모 필요 | 높음 |

| Firebase 설정 | 일부 준비 | `android/app/google-services.json`, `ios/Runner/GoogleService-Info.plist`, `lib/firebase_options.dart` 있음 | 출시 앱 ID와 패키지/번들 ID 변경 시 Firebase 앱 재등록 필요 | 높음 |

| README | 미준비 | 기본 Flutter 템플릿 문구만 있음 | 공개 저장소/협업용 설명으로 정리 필요. 스토어 제출 필수는 아님 | 낮음 |

## 바로 해야 할 일

| 순서 | 작업 | 이유 |
|---:|---|---|
| 1 | 깨진 한글 문구 복구 | 권한 팝업, 개인정보 문서, 스토어 문구가 깨지면 심사와 사용자 신뢰에 바로 영향 |
| 2 | Android `applicationId`, iOS Bundle ID 최종 확정 | Firebase, Google Maps, 스토어 등록, 서명 설정의 기준점 |
| 3 | Android release signing 설정 | 현재 debug signing이라 Google Play 제출 불가 |
| 4 | 개인정보처리방침 본문과 공개 URL 준비 | Google/Apple 모두 사실상 필수 |
| 5 | Data Safety / App Privacy 작성표 만들기 | 위치 기반 앱이라 심사 리스크가 가장 큼 |
| 6 | iOS 테스트 리포트의 긴급 이슈 해결 | iOS 제출 전 안정성 차단 요소 |
| 7 | 스토어 스크린샷 6장 세트 제작 | 현재 후보는 있으나 제출 규격과 메시지 설계가 부족 |
| 8 | 피처 그래픽과 512px 아이콘 제작 | Google Play 등록 이미지 필수/핵심 자산 |
| 9 | 심사자 메모와 테스트 절차 작성 | 가입 없는 앱, 위치 권한 앱일수록 심사자가 이해하기 쉽게 해야 함 |

## 제출 전 최종 확인 질문

| 질문 | 현재 판단 | 필요 결정 |
|---|---|---|
| 스토어 표시명을 `Z:GUM`으로 할까, `ZGUM`으로 할까? | 앱 내부는 `Z:GUM`, 플랫폼 표시명은 `ZGUM` | 하나로 통일 |
| 백그라운드 위치를 쓸까? | 현재 명시 권한은 없음 | 이음/근접 알림의 실제 동작 방식 확정 |
| 기기 인증을 앱 시작 때 필수로 둘까? | 현재 `AuthGateScreen`이 시작 시 인증 요청 | 가입 없는 앱 정책과 사용자 진입 UX 기준으로 재검토 |
| 파트너/관리자 기능을 심사자가 어떻게 확인할까? | 기능은 있으나 심사용 접근 방법은 불명확 | 심사 메모 또는 테스트 절차 작성 |
| 공식 연락 이메일은 무엇으로 제출할까? | 기존 메모상 `zigumproject@gmail.com` 후보 | 공개 제출 전 최종 확인 |
