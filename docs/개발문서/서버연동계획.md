# 서버 연동 계획

## 현재 상태

- 모든 데이터 SharedPreferences 로컬 저장
- 공공 API (TourAPI·KOPIS·SDSC·Kakao Local) 연동 완료
- Firebase 구조 설계 완료, 연동 미완

## Firebase 연동 목록

| 항목 | 내용 | 우선순위 |
|---|---|---|
| 인증 | 외부 본인인증 서비스 연동 (PASS 등) | 높음 |
| 체크인 저장 | Firestore — 사용자 흔적 클라우드 저장 | 높음 |
| 파트너 이벤트 | Firestore — 이벤트 등록·조회·만료 | 높음 |
| 알림 저장 | Firestore — 파트너 알림 수신·읽음 처리 | 높음 |
| 친구 시스템 | Firestore — 이음 신청·수락·목록 | 중간 |
| 관심 기능 | Firestore — 이벤트 저장·시작 알림 | 중간 |
| 이벤트 환불 | Functions — 10% 이내 종료 크레딧 처리 | 낮음 |

## 결제 연동

- Naver Pay·Kakao Pay 연동 예정
- 이벤트 등록 시 노출시간 기반 과금
- 무료이용 기간(6개월) 종료 후 적용

## 인증 전략

- 앱 내 개인정보 입력창 금지
- 외부 본인인증 서비스 (PASS 등) 연동
- 인증 비용 발생하더라도 구조적 안전 우선

## Firestore 보안 룰

- 사용자는 본인 데이터만 읽기·쓰기
- 파트너 이벤트는 인증된 파트너만 등록
- 관리자 전용 컬렉션 별도 분리
