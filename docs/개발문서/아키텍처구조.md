# 아키텍처 구조

## 기술 스택

- **프레임워크**: Flutter (Android/iOS 크로스플랫폼)
- **상태 관리**: Riverpod
- **지도**: flutter_map + OpenStreetMap (카카오맵 전환 준비)
- **로컬 저장**: SharedPreferences (Firebase 전환 예정)
- **인증**: local_auth (생체 인증)
- **언어**: Dart

## 계층 구조

```
[공공 API 계층]
TourAPI / KOPIS / SDSC / Kakao Local API
        ↓
[Repository 계층] — 인터페이스 + Mock/API 구현체 분리
        ↓
[Provider 계층] — Riverpod 상태 관리
        ↓
[UI 계층] — Flutter (Android/iOS 동시 빌드)
        ↓
[로컬 저장] — SharedPreferences (클라우드 전환 예정)
```

## 폴더 구조

```
lib/
├── core/           — 공통 Provider·인터페이스·테마·상수
├── data/           — 모델·Repository·어댑터
├── features/       — 기능별 독립 모듈
│   ├── map_room/   — 지도 엔진·화면
│   ├── partner_room/ — 파트너 등록·대시보드
│   ├── user_room/  — 사용자 페이지·설정
│   └── alert/      — 알림·지오펜스
├── presentation/   — Shell(탭 구조)·공통 위젯
├── promotions/     — 한시적 행사 코드 (무료이용 등)
└── services/       — 위치·알림 서비스
```

## 핵심 기술 구현

- **GPS 4단계 폴백**: GPS 정밀 → 이전 세션 위치 → Wi-Fi/기지국 → 기본값(서울시청)
- **행사 시간 판정**: 0~60분(긴박 펄스), 30분~3시간(추천 표시), 30분 미만(표시 제외)
- **어댑터 패턴**: 공공 API 응답을 통합 AppModel로 변환 — API 교체 시 어댑터만 수정
- **지도 엔진 인터페이스**: MapEngine 인터페이스로 flutter_map·카카오맵 교체 가능
- **패널 UX**: Transform.translate + RepaintBoundary — 패널 애니메이션이 지도 repaint 비유발
