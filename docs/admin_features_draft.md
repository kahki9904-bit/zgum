# Z:GUM 관리자 기능 설명서

(2026-07-02 업데이트 — 역할 개념 재정리 세션 결과 반영)

---

## 역할 구분 (통칭 금지)

"관리자"를 통칭해서 부르면 오해가 생겨서 아래처럼 명확히 나눔.

| 역할 | 설명 |
|---|---|
| **총괄** | 지도 검색창 13자리 숫자로 이스터에그 진입, 단말기 자체가 보안키. 앱 전체 권한 |
| **관리자** | 총괄이 이음(직접 만남)으로 지정, 웹에서 QR 인증으로 접근 |
| **파트너** | 관리자 아님. by Z:GUM(개별 사업장 계약) 또는 Z:GIM ZONE(기관 협업)으로 별도 계약한 이용자 |
| **등록자(사용자)** | 관리자 아님. 이벤트에 등록/체크인해서 이용하는 일반 이용자 |

---

## 지정 흐름

1. **총괄 인증** — 지도 검색창에 특정 13자리 숫자 입력 → 해시 일치 시 그 단말기가 총괄로 등록. 새 기기에서 인증하면 이전 기기 자동 해제
2. **관리자 지정** — 총괄이 상대방과 실제로 만나서 "이음" 연결하면 자동으로 관리자 등록. 계정·비밀번호 없이 "실제로 만난 것" 자체가 인증
3. **해제** — 이음 기간 만료 시 자동 해제, 또는 총괄이 목록에서 수동 해제

**웹 접근(QR 로그인):** 계정·비밀번호 없음. 웹페이지에 뜬 QR을 관리자 폰 카메라로 찍어서 인증 → 단말기가 곧 보안키.

---

## 권한 항목 (12개)

관리자는 기본으로 8개 전부 자동 부여됨. 총괄이 필요시 특정 관리자의 특정 항목만 골라서 제외 가능 (opt-out 방식).

| 항목 | 총괄 | 관리자 | 비고 |
|---|:---:|:---:|---|
| 관리자 지정·해제 | O | | 다른 관리자를 임명/해제하는 권한이라 총괄 전용 |
| 앱 전체 긴급 차단 | O | | 서비스 전체 일시중단, 총괄 전용 |
| 알림 종류별 끄기 | O | | 전체 사용자 대상 알림 종류별 on/off, 총괄 전용 |
| API 한도 확인 | O | | 카카오/공공API 등 사용량 확인, 총괄 전용 |
| 활동 기록 조회 | O (전체) | O (본인 것만) | 감사 로그 |
| 필터 정책 변경 | O (전체) | O (본인 범위) | 콘텐츠 필터 기준, 실시간 반영 |
| 계정 차단·이용정지 | O | O | 사용자/파트너 계정 상태 관리 |
| 이벤트 강제 종료 | O | O | 부적절한 이벤트 즉시 종료 |
| 문의·의견 대응 | O | O | 사용자 신고/문의 응대 |
| 결제(과금) 현황 조회 | O (전체) | O (본인 관리 파트너만) | 초기엔 무료이용, 이후 과금 전환 대비 |
| 파트너 이메일 등록 + 무료이용 기간 부여 | O | O | 이메일 인증 작업 위에 얹어서 구현 예정 |
| 앱 강제 업데이트 | O | O | 구버전 사용자만 차단, 최신버전은 정상 이용 (긴급차단과 별개) |

## 화면 진입 방식

3페이지(파트너 방) 오른쪽 상단에 막대그래프 모양 배지가 총괄/관리자로 지정된 사람에게만 나타남. 일반 사용자·파트너에게는 절대 비노출.

- 총괄이 탭하면 → 12개 항목 전부 노출
- 관리자가 탭하면 → 그중 8개만 노출 (총괄 전용 4개 제외)

---

## 구현 순서 (6단계)

1. **스위치 켜기** — 꺼져있는 총괄/관리자 로직 활성화, 이음 완료 시 실제 지정되도록 연결
2. **관리자 진입 배지** — 3페이지 배지 생성, 탭하면 관리자 메뉴로 이동 (처음엔 빈 메뉴여도 됨)
3. **총괄 전용 화면** — 관리자 목록/해제 → 긴급차단 → 알림종류별끄기 → API한도 순
4. **관리자 실무 기능** — 활동기록·필터정책·계정차단·이벤트강제종료·문의응답
5. **공통** — 파트너 공간 관리 (by Z:GUM 연결)
6. **웹 QR 로그인** — 가장 나중, 공수 최대 (웹+앱+Firebase 세션 동기화 전부 신규)

## 보류 항목

- **Z:GIM ZONE 데이터 등록/관리** — 스토어 출시 우선이라 나중으로 미룸
- **무료이용 행사 현황 조회** — 임시 프로모션 기능이라 우선순위 낮음

---

## 전략 원칙

데이터 분석/대시보드는 앱의 핵심 가치가 아님. 관리자 화면에 통계·그래프 기능을 과도하게 넣지 않는 방향으로 설계. (관련: 등록자/파트너용 데이터 대시보드 방향은 전면 폐기, 방문자수·흔적수 두 개만 최소한으로 노출)
